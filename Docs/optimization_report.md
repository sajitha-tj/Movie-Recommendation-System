# Optimization Report

## Introduction

This report outlines the optimizations implemented in the Movie Recommendation System, focusing on improving the performance of hotspots identified in the program using profiling tools ([`perf`](https://perfwiki.github.io/main/)). The optimizations were aimed at reducing the execution time and enhancing the efficiency of the algorithms used.

## Methodology

### Initial Code Review and Fixes

The intial [Movie-Recommendation-System](https://github.com/shubham-bhatnagar/Movie-Recommendation-System) program contained some legacy code and problems that were identified and fixed to ensure the program runs correctly. The fixed code can be found in the [fixed](https://github.com/sajitha-tj/Movie-Recommendation-System/tree/fixed) branch.

### Baseline Performance Measurement

Initial program was heavily dependent on user input, which was not ideal for performance testing. To address this, the program was modified to run with predefined inputs, allowing for consistent performance measurements. Modified code can be found in the [baseline](https://github.com/sajitha-tj/Movie-Recommendation-System/tree/baseline) branch. This baseline version was then executed for 5 consecutive runs and following are the time taken for recommendation in each run:

The average time taken for recommendation in the baseline version was calculated to be approximately **0.152 seconds**.

### Optimization Strategies

#### 1. Optimizing `calc_similarity`

The original implementation of `calc_similarity` introduced significant overhead by dynamically allocating memory for a temporary array `A` inside the loop for every user, copying an entire row of the normalized matrix into it, and then freeing it. This resulted in repeated heap allocations and deallocations, which are relatively expensive operations, as well as unnecessary memory copying proportional to the number of users multiplied by the number of movies. These operations not only increased execution time but also negatively impacted cache performance, since data was being duplicated instead of accessed directly from its original contiguous location in memory.

The optimized version eliminates these inefficiencies by directly referencing each user’s row in the normalized matrix using pointer arithmetic, avoiding both dynamic memory allocation and redundant copying. By accessing the data in-place, the function significantly reduces memory traffic and improves cache locality, allowing the CPU to process data more efficiently. This change leads to a noticeable performance improvement, especially for large datasets, and also enables further optimizations such as compiler vectorization and parallel execution, since the loop iterations are independent and operate on contiguous memory regions.
